## 当前AI记忆的痛点

AI助手在对话之间会遗忘一切。现有的解决方案——向量数据库、RAG管道、记忆插件——都存在相同的根本问题：

| 痛点 | 具体表现 |
|---|---|
| **不可见** | 记忆被埋在向量数据库中，你看不到存了什么、事物之间如何关联、哪里出了问题 |
| **不可控** | 无法编辑、合并、分类或删除单条记忆，完全是个黑盒 |
| **不自维护** | 记忆堆积产生重复、矛盾和过时信息，没有人清理 |
| **不自进化** | 系统只知道你明确告诉它的内容，从不主动学习 |

MindReader V2 解决了所有这些问题。

## 什么是 MindReader V2？

MindReader 是一个**个人知识图谱**，能够从AI对话中捕获、组织和回忆信息。不同于扁平化的记忆存储，它构建了一个由实体和关系组成的结构化图谱——并提供完整的可视化界面让你查看和管理一切。

### 核心理念

- **全面可视** — 交互式图谱可视化展示你的整个记忆全景
- **完全掌控** — 编辑实体、管理标签、合并重复项、手动创建关系
- **自动维护** — LLM驱动的自动分类、自动打标、关系修复和去重
- **自我进化** — 节点通过网络搜索自主研究和扩展自身知识

## 功能特性

### 自我进化 — 会自己生长的记忆

这是MindReader与其他所有AI记忆系统的根本区别。你的知识图谱不仅仅存储你告诉它的内容——**它会主动研究和自我扩展**。

选中任意节点，点击 **Evolve（进化）**。MindReader会将该实体发送给具备网络搜索能力的LLM：

1. **研究** — 通过实时网络搜索查找最新的事实、关联和上下文
2. **发现新实体** — 自动找到与原始节点相关的人物、组织、项目、概念
3. **建立关系** — 自动创建带类型的连接（works_at、developed_by、part_of 等）
4. **实时流式呈现** — 在交互式图谱上实时观看新节点和边的出现
5. **保存到图谱** — 审查发现的内容，然后一键保存到你的知识库

进化图谱会将已有节点（蓝色）和新发现的实体（按类别着色）并排展示，让你在提交之前清楚看到LLM发现了什么。

https://github.com/user-attachments/assets/4a93cef0-e48e-464d-a70c-1759d16c5de2

> *单次进化通常能发现10-25个新实体及其关系，将一个孤立节点变成丰富的知识集群。*

这意味着你的记忆图谱会**复利增长** — 每次进化产生的新节点本身又可以被进化，构建一个不断扩展的知识网络，无需你手动输入任何内容。

### 可视化知识图谱

以交互式力导向图谱探索你的记忆。节点代表实体（人物、项目、工具、地点），边代表关系和事实。

- 缩放、平移、点击探索
- 节点大小按连接数量自动调整——重要实体更大
- 按类别颜色编码（人物、项目、公司等）
- 悬停快速预览分类、标签和摘要
- 按类别过滤，专注重点内容

### 实体管理

点击任意实体打开详情面板，完全掌控你的记忆：

- **标签** — 添加描述性标签（LLM自动提取，支持手动编辑）
- **分类** — 12个内置分类，完全可自定义
- **摘要** — 点击即可编辑的摘要，支持AI生成解读
- **进化** — 通过网络搜索研究和扩展任意节点的知识
- **关联** — 手动创建实体间的关系
- **合并** — 合并重复实体，保留所有关系
- **删除** — 删除前预览影响范围（显示受影响的关系）

### 智能搜索

通过统一搜索栏（Ctrl+K）跨实体名称、摘要和标签搜索。

- **标签感知搜索** — 搜索"swimmer"能找到标记了"swimmer"的实体，即使名称中没有这个词
- **上下文感知** — 在图谱标签页中，搜索会高亮并缩放到匹配的节点
- **相关性排序** — 精确匹配 > 前缀匹配 > 包含匹配 > 标签匹配

### 自动维护

MindReader不只是存储记忆——它还能保持记忆的整洁。

| 功能 | 工作方式 |
|---|---|
| **自动分类** | LLM根据名称、摘要和标签自动分类未归类实体 |
| **自动打标** | 在同一次LLM调用中提取描述性标签（角色、技能、地点、关系） |
| **批量重分类** | 一键处理未归类实体，支持配置批次大小 |
| **重复检测** | 扫描名称和属性相似的实体 |
| **关系修复** | 检测方向错误、拼写错误和模糊关系（规则 + LLM双引擎） |
| **孤立清理** | 发现并移除断开连接的实体 |

![维护中心](docs/images/memory%20automastically%20clean%20up.png)
*维护中心——使用AI批量重分类未归类实体。LLM分析每个实体的名称和摘要，自动分配最合适的分类。*

### 命令行工具

为高级用户和自动化提供完整的命令行界面：

```bash
mg search "游泳比赛"                    # 语义搜索，附带实体档案
mg search "Aria" --json               # 机器可读的JSON输出
mg tags "Aria Lu"                     # 查看标签: Aria Lu [person]: swimmer, daughter
mg tags "Aria Lu" --add "competitive" # 添加标签
mg tags --backfill                    # LLM自动为所有实体提取标签
mg add "Alice是一名数据科学家"          # 存储新记忆
mg entities --limit 20                # 列出实体
mg maint scan                         # 扫描问题
mg maint fix                          # 自动修复重复和孤立项
```

### OpenClaw 集成 — 聊天即建库

MindReader 是 [OpenClaw](https://github.com/openclaw) 的一等公民扩展。安装一次，之后你与AI代理的每一次对话都会自动构建和维护你的个人知识图谱——无需任何手动操作。

- **自动捕获** — 当你与代理对话时，MindReader在后台默默提取实体、事实和关系，存入你的知识图谱
- **自动回忆** — 代理回复之前，相关记忆会自动检索并注入上下文，让代理拥有跨会话的长期记忆
- **自我净化** — 后台维护持续运行：自动分类新实体、提取标签、检测重复、修复关系——你的记忆始终保持整洁，无需任何干预
- **工具调用** — 代理还可以通过MindReader的工具接口显式搜索、存储和列出记忆
- **可视化界面** — 随时访问完整的图谱浏览器，查看、管理和进化你积累的知识

> 只需自然地与你的代理对话。MindReader会处理一切——捕获重要信息、组织成结构化图谱、并持续保持整洁。

## 架构

MindReader采用npm workspaces单仓库组织：

```
mindreaderv2/
  packages/
    mindgraph/         # Python核心 - Graphiti记忆引擎、CLI、后台工作器
    mindreader-ui/     # Express服务 + React界面 - 可视化与管理
      server/
        server.js      # 应用组装（约150行）
        routes/        # 8个路由模块（graph、entity、categories、search、cleanup、audit、tokens、cli）
        lib/           # 共享工具（daemon、categorizer、LLM客户端、preprocessor预处理器）
      ui/              # React前端
    openclaw-plugin/   # 可选AI代理集成 - 自动回忆/捕获
```

### 工作原理

```
对话 ──> 捕获 ──> 预处理器 ──> Neo4j知识图谱 ──> 回忆 ──> AI上下文
          │          │                │                      │
          ▼          ▼                ▼                      ▼
      过滤消息    分类事实         自动分类             语义搜索
      提取事实    属性 → Neo4j     自动打标             实体档案
      匹配实体    关系 → Graphiti  关系修复             结构化JSON
                                       │
                                       ▼
                                   自我进化
                                  （网络搜索LLM发现
                                   新实体和关系）
```

1. **捕获** — 对话被过滤后，LLM预处理器提取并分类事实再进行存储
2. **预处理** — 每条事实被分类为*属性*（直接写入Neo4j实体的标签/摘要）或*关系*（转发给Graphiti进行图谱存储）。这避免了"开发者"或"15年经验"这类垃圾实体污染图谱。
3. **组织** — LLM持续自动分类、自动打标并维护图谱
4. **进化** — 任意节点可通过网络搜索驱动的研究进行扩展，发现新实体和关系
5. **回忆** — 语义搜索检索相关记忆，附带完整实体上下文

### 智能存储 — 质量优于数量

传统知识图谱系统把一切都当作实体处理。告诉它"Dell是一个拥有15年经验的开发者"，就会创建三个实体："Dell"、"开发者"和"15年经验"。后两者是属性，不是独立实体——它们会污染图谱。

MindReader的**预处理管道**解决了这个问题：

1. **已知实体查找** — 分类前，预处理器在Neo4j中搜索文本中提到的实体
2. **LLM分类** — 每条事实被分类为*属性*（角色、技能、特征、偏好 → 直接更新Neo4j）或*关系*（实体间的连接 → Graphiti）
3. **直接属性写入** — 标签和摘要更新直接写入现有实体节点，避免不必要的图谱遍历
4. **优雅降级** — 如果预处理失败，系统会回退到Graphiti并附带自定义提取指令，仍然能防止垃圾实体创建

手动存储路径（`memory_store`工具）和自动捕获路径（对话结束钩子）都经过此管道。自动捕获路径还会在分类前从对话历史中提取关键事实，过滤掉代码、调试输出和工具结果。

可以通过`PREPROCESS_MODE`环境变量切换预处理模式：

| 模式 | 行为 | LLM调用次数 |
|------|------|-------------|
| `merged`（默认） | 一次LLM调用完成事实提取+分类 | 1 |
| `two-pass` | 分开的提取和分类步骤 | 1 + N |

## LLM支持

| 提供商 | 状态 | 默认模型 | 网络搜索 |
|---|---|---|---|
| OpenAI | 已支持 | `gpt-4o-mini` | 否 |
| DashScope（阿里巴巴） | 已支持 | `qwen3.5-flash` | 是（内置） |
| Anthropic | 已支持（通过代理） | 需要OpenAI兼容代理 | 取决于代理 |

> 自我进化功能使用支持内置网络搜索的模型（如DashScope的Qwen系列）效果显著更好。

## 配置

配置存储在仓库根目录的 `.env` 文件中。安装向导会自动生成，也可手动配置：

```bash
cp .env.example .env
```

关键变量：`LLM_PROVIDER`、`LLM_API_KEY`、`NEO4J_URI`、`NEO4J_USER`、`NEO4J_PASSWORD`。
